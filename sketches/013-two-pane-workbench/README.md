## Variant: Two-pane Workbench

### Design stance
Treat the product like an IDE or note editor: an idea index on the left, the selected idea as a full editable document on the right.

### Key choices
- Layout: two panes only; no inspector rail. Metadata, AI, and files live inside the selected document.
- Typography: large document title, small mono section labels, compact index cards.
- Color: current dark/purple palette with restrained surfaces and borders.
- Interaction: selecting cards updates the document; AI block expands inline.

### Trade-offs
- Strong at: long-term maintainability, writing depth, clear board/detail relationship.
- Weak at: classic Kanban spatial flow is reduced unless status filters/actions are strong.

### Best for
- A durable personal developer tool where ideas become documents, specs, and agent handoff packets.
