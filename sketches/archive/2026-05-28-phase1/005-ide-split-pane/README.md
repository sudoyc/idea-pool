## Variant: IDE Split-Pane

### Design stance

Long-term personal workspace/library. This is closest to Gemini `3.html` and treats ideas as documents, not cards.

### Key layout choices

100vh two-pane app shell, fixed compact input sidebar, independently scrolling idea list, and a wide reading pane for the selected idea.

### Typography choices

Compact 12-14px sidebar text, 31-42px detail title, 20px section headings, 15.5px body copy with generous line height, and monospace only for metadata/action snippets.

### Interactions

Select ideas in the sidebar to change the document, regenerate the list from controls, and copy the selected outline with toast feedback.

### Strong at

Durable V1 structure, reading comfort, visible input-output relationship, and future extensibility.

### Weak at

Less playful than deck/feed concepts and needs careful mobile adaptation.

### Best for

The production app if it should feel like a personal developer workspace.
